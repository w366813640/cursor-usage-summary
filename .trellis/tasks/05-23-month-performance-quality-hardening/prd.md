# Week 4: Performance and Quality Hardening

## Goal

Turn the recent UX work into a maintainable release train with measurable bundle output, focused tests, and accessibility smoke coverage.

## Requirements

- Add a repeatable bundle budget/report command or script.
- Verify lazy route chunks remain split.
- Add focused tests around new planning/personalization logic.
- Document manual accessibility smoke checks.

## Acceptance Criteria

- A developer can run the quality checks locally.
- Build output highlights route chunks and initial bundle size.
- Tests cover new pure logic.
- Typecheck, lint, and relevant tests pass.
